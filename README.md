# PUBG Tactical Hub (绝地求生战术助手)

PUBG Tactical Hub 是一个基于屏幕截图、OpenCV 模板识别和系统级键鼠监听/模拟实现的 PUBG 战术辅助工具。项目不读取游戏内存、不注入 DLL、不修改游戏文件，核心能力来自对游戏画面的实时视觉分析、区域校准和弹道/距离计算。

> 本项目仅供 Python 计算机视觉、桌面 UI、游戏画面识别和自动化技术学习交流。使用第三方工具或宏功能可能违反游戏发行商规则，请自行承担使用风险。

## 当前能力

- **地图测距**：支持小地图实时测距、大地图快照测距、垂直测高和迫击炮距离修正。
- **地图点位**：支持显示载具、飞机、密室、安全门、特殊房间等战术资源点位。
- **特殊武器辅助**：支持迫击炮、火箭筒、投掷物、VSS、十字弩、C4 等显示和弹道计算。
- **武器与配件识别**：通过模板匹配识别当前武器、装备栏武器、倍镜、握把、枪口、枪托和姿势。
- **辅助压枪**：根据当前武器、配件、姿势和配置曲线计算压枪力度。
- **可视化校准**：提供小地图、大地图、垂直测高、武器栏、装备栏配件、姿势区域、1km 标尺等校准入口。
- **配置驱动**：快捷键、截图区域、模板阈值、色盲模式、弹道数据、压枪曲线和地图点位均集中在 `config.json`。

## 项目结构

```text
.
├── main.py                  # 主程序入口，Tkinter UI 与模块调度
├── config.json              # 区域、快捷键、弹道、压枪、地图点位等配置
├── modules/                 # 功能模块与调试/校准工具
├── templates/               # 武器、配件、姿势、标点、倍镜等识别模板
├── 使用说明.md              # 更完整的使用流程与故障排查
├── icon.ico                 # 程序图标
└── recoil.lua               # 压枪相关脚本/参考配置
```

## 主要模块

- `minimap_radar.py`：小地图标点检测与实时距离计算。
- `largemap_radar.py`：大地图快照测距。
- `elevation_radar.py`：垂直测高，用于高低差判断。
- `mortar_assistant.py`：迫击炮测距与高低差修正。
- `map_assistant.py`：地图资源点位显示。
- `weapon_detector.py`：当前手持武器识别。
- `equipment_detector.py`：装备栏武器与配件识别。
- `gesture_identifier.py`：站立、蹲下、趴下姿势识别。
- `recoil_control.py`：辅助压枪控制。
- `throwables_assistant.py`：投掷物抬高与瞬爆控制。
- `rocket_assistant.py` / `vss_assistant.py` / `crossbow_assistant.py` / `c4_assistant.py`：特殊武器辅助。
- `region_manager.py`：全局截图区域、比例尺和调试覆盖层管理。
- `region_calibrator.py` / `region_calibrator_auto.py`：区域与缩放校准工具。
- `recoil_debugger.py` / `special_weapon_debugger.py`：压枪和特殊武器调试窗口。

## 运行环境

推荐环境：

- Windows 11
- Python 3.10.x
- PUBG 使用全屏或无边框全屏
- 游戏分辨率建议 `1920x1080`
- 游戏语言建议中文
- 更换分辨率、显示器、游戏 UI 缩放后需要重新校准

安装依赖：

```bash
pip install opencv-python numpy mss pynput Pillow
```

运行主程序：

```bash
python main.py
```

> 代码中包含较多 Windows API 调用，完整功能主要面向 Windows 环境。

## 首次使用流程

第一次使用建议先完成校准，再开启辅助功能。

1. 启动游戏并进入训练场或对局。
2. 启动 `main.py`。
3. 进入 `校准区域` 选项卡。
4. 依次校准 `小地图`、`大地图`、`垂直测高`、`武器栏`。
5. 校准 `武器1` 和 `武器2` 的编号、名称、倍镜、握把、枪口、枪托。
6. 校准 `姿势区域`。
7. 打开大地图，校准 `1km长度`。
8. 完成后重启助手，让配置稳定生效。

如果只使用地图点位或特殊武器显示，可以先校准小地图、大地图、垂直测高和 1km 长度；如果要使用压枪，建议完整校准武器、配件和姿势区域。

## 日常使用流程

1. 启动游戏并进入对局。
2. 启动助手。
3. 在 `地图点位` 中选择当前地图、标点大小、色盲模式和点位类型。
4. 按 `F1` 开启武器检测。
5. 按 `Tab` 打开装备栏，让助手识别两把主武器和配件。
6. 按 `F2` 开启瞄准辅助显示层。
7. 需要压枪时按 `F3` 开启辅助压枪。
8. 需要大地图测距时打开大地图并按 `F4`。

## 主窗口选项卡

- `地图点位`：选择地图、点位大小、色盲模式和点位类型。
- `启动助手`：开启武器检测、瞄准辅助、辅助压枪和特殊武器助手。
- `校准区域`：校准所有截图识别区域、比例尺和调试显示。
- `按键设置`：查看、修改、保存或恢复快捷键。

## 默认快捷键

| 功能 | 默认快捷键 | 说明 |
| --- | --- | --- |
| 显示 / 隐藏主窗口 | `Home` | 固定快捷键，不在按键设置里修改。 |
| 武器检测开关 | `F1` | 开启或关闭基础识别。 |
| 瞄准辅助开关 | `F2` | 开启或关闭测距与特殊武器显示层。 |
| 辅助压枪开关 | `F3` | 开启或关闭压枪模块。 |
| 大地图测距 | `F4` | 打开大地图后触发测距流程。 |
| 投掷物瞬爆 | `V` | 按下准备，松开投掷。 |
| 上一个标点颜色 | `Q` | 切换当前标点颜色。 |
| 下一个标点颜色 | `E` | 切换当前标点颜色。 |
| 装备栏识别触发 | `Tab` | 打开装备栏后识别武器和配件。 |

快捷键可以在 `按键设置` 选项卡中修改并保存到 `config.json`。

## 配置文件说明

`config.json` 是项目的核心数据文件，主要包含：

- `hotkeys`：可配置快捷键。
- `real_regions`：用户校准后的真实截图区域。
- `real_scales`：用户校准后的比例尺。
- `region_scaling_settings`：部分识别区域的基准尺寸。
- `pnt_color_mode` / `pnt_color_modes` / `pnt_colors`：标点颜色与色盲模式阈值。
- `throwables_config`、`rocket_config`、`vss_config`、`crossbow_config`、`c4_config`、`mortar_config`：特殊武器弹道和时间参数。
- `recoil_settings`：压枪按键、延迟、曲线、武器类型和配件倍率。
- `map_data`：各地图资源点位数据。

不建议手动删除配置字段。若识别异常，优先通过程序里的校准区域重新框选。

## 模板资源

`templates/` 中保存 OpenCV 模板匹配所需图片：

- `templates/weapons/`：手持武器识别模板。
- `templates/equipments/`：装备栏武器名、编号、倍镜、握把、枪口、枪托模板。
- `templates/gestures/`：站立、蹲下、趴下模板。
- `templates/pnt/`：小地图、大地图、垂直测高标点模板。
- `templates/scopes/`：倍镜识别模板。

模板文件名和目录结构会被代码直接读取，不建议随意改名或移动。

## 打包

Windows 打包命令：

```bash
nuitka --onefile --plugin-enable=tk-inter --windows-disable-console --windows-icon-from-ico=icon.ico --include-data-files=icon.ico=icon.ico main.py
```

如果打包后模板识别或配置读取异常，请确认运行目录中包含 `config.json` 和 `templates` 文件夹，或根据 Nuitka 的数据文件规则额外包含这些资源。

## 常见问题

**识别不到武器或配件**

打开装备栏后再触发识别，检查武器名称、编号、倍镜、握把、枪口、枪托区域是否校准准确。

**小地图或大地图距离不准**

重新校准小地图、大地图和 1km 长度，并确认游戏色盲模式与助手里的色盲选择一致。

**特殊武器显示层不出现**

确认 `F2` 瞄准辅助已开启，当前武器识别正确，并且小地图内存在可识别标点。

**压枪效果异常**

确认 `F1` 武器检测已开启，装备栏配件识别正确，姿势区域校准准确，并检查 `config.json` 中对应武器的压枪配置。

**快捷键没反应**

检查是否正在录制快捷键、是否与游戏按键冲突，必要时进入 `按键设置` 保存一次或重启助手。
